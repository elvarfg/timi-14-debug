🔧 DevTools Sýnikennsla

1. Console.log() og undirstöður

Við byrjum á að skoða hvernig console virkar og hvernig við getum notað það til að debugga.

Niðurstöður birtast í Console (F12)
💡 Sýna: Opnaðu Console og sýndu hvernig mismunandi console aðferðir birtast. Útskýrðu muninn á log, warn, error og table.

2. Elements flipinn - HTML/CSS breytingar

Hér getum við sýnt hvernig Elements flipinn virkar til að skoða og breyta HTML/CSS.

Jón Jónsson

Forritari

💡 Sýna:
  • Hægri-smella á element → Inspect
  • Breyta texta beint í Elements
  • Bæta við/fjarlægja CSS í Styles
  • Sýna computed styles
  • Sýna box model

3. Breakpoints og Step Debugging

Lærum að nota breakpoints til að stöðva keyrslu og skoða gildi.

Niðurstaða birtist hér
💡 Sýna:
  • Setja breakpoint á línu í Sources
  • Nota debugger; statement
  • Skoða Scope panel (Local, Global)
  • Step over, Step into, Step out
  • Watch expressions

4. Finna og laga villur

Við skoðum algeng villudæmi og hvernig Console hjálpar okkur að finna þau.

Villur birtast í Console
💡 Sýna: Smelltu á takkana og sýndu hvernig villurnar birtast í Console. Útskýrðu hvernig á að lesa villuna og finna línunúmer.

5. Event Listeners og DOM manipulation

Skoðum hvernig við finnum event listeners og debuggum DOM breytingar.

  • Læra JavaScript
  • Æfa DevTools
💡 Sýna:
  • Event Listeners flipinn í Elements
  • Hvernig DOM breytist í Elements þegar við bætum við
  • Break on subtree modifications

6. Network flipinn

Skoðum hvernig við fylgjumst með netbeiðnum og API köllum.

Gögn birtast hér
💡 Sýna:
  • Network flipann og beiðnirnar
  • Status codes (200, 404, 500)
  • Preview og Response flipana
  • Timing upplýsingar
  • Filter eftir tegund (XHR, JS, CSS)

7. Conditional Breakpoints og Logpoints

Fullkomnari debugging tækni fyrir lykkjur og endurtekin köll.

Niðurstöður birtast hér
💡 Sýna:
  • Hægri-smella á breakpoint → Add conditional breakpoint
  • T.d. stoppa aðeins ef i === 15
  • Logpoint: prentar án þess að stoppa
